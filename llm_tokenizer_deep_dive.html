The Complete Pipeline — What Really Happens
1
📖
Pre-built Vocabulary
The tokenizer vocab is frozen before training. GPT-4 has 100,277 tokens. Llama 3 has 128,000. IDs never change after training.
"hello" → always ID 15339 in GPT-2
2
✂️
BPE Splitting
Byte Pair Encoding merges the most frequent character pairs repeatedly until a vocabulary is built. Rare words get split into subword pieces.
"tokenization" → ["token","ization"]
3
🔢
ID Lookup
Each token piece is looked up in the vocabulary table. The result is a flat list of integers — the model never sees letters.
"hi there" → [5303, 612]
4
📐
Embedding Lookup
Each ID maps to a vector of 768–4096 numbers from a learned embedding table. This vector encodes the token's meaning.
ID 5303 → [0.23, -0.87, 0.41 ...]
5
👁️
Attention — Reuse Matters
Every token attends to every other. Repeated token IDs create shared attention pathways — the model learns their context relationship.
"the" × 10 → strong positional patterns
6
🎯
Prediction
The transformer outputs a probability over all 100K vocab tokens for the next token. The highest probability ID is selected and decoded back to text.
→ softmax → ID 42 → "world"

Raw Sequence
Byte Pair Encoding (BPE) — the tokenizer starts with individual characters and repeatedly merges the most frequent adjacent pair. After ~50,000 merges, it has a vocabulary of common words and subword pieces. Rare or unknown words get split into familiar subword pieces — nothing is ever truly "unknown" to a BPE tokenizer.
Each colored block is one real BPE token. Same color = same token ID across the entire text. The number under each token is its actual ID from the model vocabulary — hover to see details. Notice how common words like the, is are single tokens, while rare words get split.
After token IDs are assigned, each ID is looked up in an embedding table — a matrix of shape [vocab_size × embed_dim] (e.g. 50,257 × 768 for GPT-2). Each token becomes a vector of 768 numbers encoding its meaning. Similar tokens have similar vectors — this is how the model understands "king" ≈ "queen".
The attention mechanism computes a score between every pair of tokens. When token ID X repeats, those positions develop shared attention patterns — the model learns that these tokens are contextually linked.
