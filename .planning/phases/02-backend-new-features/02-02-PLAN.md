  <action>
    1. **Add tiktoken to `requirements.txt`**:
       - Add `tiktoken` (latest version) to requirements.txt. Install it.

    2. **Update `models.py`**:
       - Update `TokenizeRequest.model` Literal to include 'gpt4': `Literal["gpt2", "bert-base-uncased", "gpt4"]`
       - Add `context_window_usage: Optional[Dict[str, float]] = None` field to `TokenizeResponse`. Import `Dict` from typing.
       - The dict maps context window names to usage percentages. Example: `{"gpt2": 0.195, "gpt35": 0.049, ...}`

    3. **Update `config.py`**:
       - Add "gpt4" to `supported_models` default list: `["gpt2", "bert-base-uncased", "gpt4"]`

    4. **Update `tokenizer.py`** to handle 'gpt4' model:
       - Import tiktoken at top: `import tiktoken`
       - In `TokenizerService`:
         - Add `self.tiktoken_encoder = None` in `__init__`
         - Add to `preload_models()`: load tiktoken cl100k_base encoding (`tiktoken.get_encoding("cl100k_base")`)
         - Add to `get_tokenizer()`: for "gpt4", return the tiktoken encoder (note: tiktoken Encoding is NOT a HuggingFace tokenizer — it has different API: `.encode()`, `.decode()`, `.decode_single_token_bytes()`)
       - In `tokenize_process()`:
         - Add a branch for `model_name == "gpt4"`:
           - Use `tiktoken_encoder.encode(text)` to get token IDs (no add_special_tokens — tiktoken doesn't use special tokens the same way)
           - Convert IDs to token strings using `tiktoken_encoder.decode_single_token_bytes(id).decode("utf-8", errors="replace")` for each ID
           - Create `TokenInfo` objects with appropriate type detection: all tokens are "word" type for tiktoken (no subword/special distinction needed, but check if token starts with space for "word" vs doesn't for "subword")
           - `vocab_size = 100277` (cl100k_base vocab size, hardcode this as it's a fixed property)
           - `bpe_steps = []` (empty — BPE step simulation not implemented for tiktoken per BFET-03)
           - Return the same tuple format: `(token_infos, raw_ids, vocab_size, bpe_steps)`
       - In `get_token_type()` and `get_display_text()`: Add 'gpt4' branches. For GPT-4 display text: if token starts with space, keep as-is (already human-readable). For token type: tokens with leading space → "word", no leading space at position > 0 → "subword", position 0 → "word".

    5. **Update `main.py` tokenize endpoint** to compute `context_window_usage`:
       - After computing `total_tokens` in the tokenize endpoint, calculate context window usage:
         ```python
         CONTEXT_WINDOWS = {
             "gpt2": 1024,
             "gpt35": 4096,
             "gpt4": 8192,
             "gpt4_32k": 32768,
             "llama3": 131072,
         }
         context_window_usage = {
             name: round(total_tokens / window, 6)
             for name, window in CONTEXT_WINDOWS.items()
         }
         ```
       - Add `context_window_usage=context_window_usage` to the `TokenizeResponse` constructor.
       - This is computed for ALL models (not just gpt4) since it shows how many tokens the current text would consume across different model context windows.
       - Note: embeddings are only available for gpt2 (existing behavior). For gpt4, `include_embeddings` should be ignored or produce None — add the check: `if req.include_embeddings and req.model == 'gpt2':` (already exists, just verify it doesn't break with gpt4).
