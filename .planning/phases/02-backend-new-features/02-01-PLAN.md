  <name>Task 1: Attention endpoint with BERT singleton service (BFET-01)</name>
  <files>

  <action>
    1. **Create `attention.py`** with an `AttentionService` class following the same singleton pattern as `EmbeddingService` in `embeddings.py`:
       - `__init__`: Set `self.bert_model = None` and `self.bert_tokenizer = None`, detect device (cuda/cpu)
       - `preload_model()`: Load `BertModel.from_pretrained("bert-base-uncased", output_attentions=True)` and `BertTokenizer.from_pretrained("bert-base-uncased")`. Set model to eval mode. Only load if not already loaded.
       - `get_attention(text: str) -> dict`: Tokenize text with BERT tokenizer (`add_special_tokens=True`), run forward pass with `torch.no_grad()`, extract `outputs.attentions` tuple. Take layer index 6 (7th layer, 0-indexed), average across all attention heads (dim=1, then squeeze), convert to Python list of lists (n x n float matrix). Return dict with `tokens` (list of token strings from `convert_ids_to_tokens`), `attention_matrix` (the averaged n x n list), and `layer` (int, 6).
       - Note: Reuse the existing `tokenizer_service.bert_tokenizer` for tokenization if possible, BUT since AttentionService needs to run the BertModel forward pass (not just tokenize), it's cleaner to have its own BertTokenizer instance bundled with BertModel. This avoids coupling.
       - Export singleton: `attention_service = AttentionService()`

    2. **Add Pydantic models to `models.py`**:
       - `AttentionRequest(BaseModel)`: `text: str = Field(..., min_length=1, max_length=100)`, `model: Literal["bert-base-uncased"] = "bert-base-uncased"`. The max_length=100 enforces the 100-char limit per BFET-01.
       - `AttentionResponse(BaseModel)`: `tokens: List[str]`, `attention_matrix: List[List[float]]`, `layer: int`, `model_used: str`, `processing_time_ms: float`

    3. **Add `/api/attention` route to `main.py`**:
       - Import `attention_service` from `attention` module and `AttentionRequest`, `AttentionResponse` from models
       - Add `attention_service.preload_model()` to the `startup_event`
       - Create `@app.post("/api/attention", response_model=AttentionResponse)` endpoint
       - Apply rate limiting: `@limiter.limit("30/minute")`
       - Sanitize input text with `sanitize_input()`, raise 400 if empty after sanitization
       - Call `attention_service.get_attention(clean_text)`, measure processing time
       - Return `AttentionResponse` with all fields populated
       - Wrap in try/except, raise HTTPException 500 on model errors
  </action>

       - `CompareRequest(BaseModel)`: `text1: str = Field(..., min_length=1, max_length=2000)`, `text2: str = Field(..., min_length=1, max_length=2000)`, `model: Literal["gpt2", "bert-base-uncased"] = "gpt2"`
       - `CompareResponse(BaseModel)`: `text1_token_count: int`, `text2_token_count: int`, `shared_tokens: List[str]` (tokens appearing in both), `text1_unique_tokens: List[str]` (tokens only in text1), `text2_unique_tokens: List[str]` (tokens only in text2), `model_used: str`, `processing_time_ms: float`

    2. **Add `/api/compare` route to `main.py`**:
       - Import `CompareRequest`, `CompareResponse` from models
       - Create `@app.post("/api/compare", response_model=CompareResponse)` endpoint
       - Apply rate limiting: `@limiter.limit("30/minute")`
       - Sanitize both text1 and text2 with `sanitize_input()`, raise 400 if either is empty after sanitization
       - Tokenize both texts using `tokenize_process()` from `tokenizer.py` (reuse existing logic)
       - Extract token text sets: `set(t.text for t in tokens1)` and `set(t.text for t in tokens2)`
       - Compute: `shared_tokens = sorted(set1 & set2)`, `text1_unique = sorted(set1 - set2)`, `text2_unique = sorted(set2 - set1)`
       - Return `CompareResponse` with counts, shared, and unique token lists
       - Wrap in try/except, raise HTTPException 500 on errors

    3. **Add tests to `test_tokenizer.py`**:
       - `test_attention_basic()`: POST /api/attention with {"text": "Hello"}, assert 200, assert "tokens" and "attention_matrix" in response, assert matrix is square (len == len of tokens), assert all values are floats between 0 and 1
       - `test_attention_char_limit()`: POST /api/attention with text of 101 chars, assert 422
       - `test_attention_empty()`: POST /api/attention with text that becomes empty after sanitization (e.g., "\u200b"), assert 400
       - `test_compare_basic()`: POST /api/compare with {"text1": "Hello world", "text2": "Hello there"}, assert 200, assert "shared_tokens" contains "Hello" (or the GPT2 token text), assert text1_unique and text2_unique are non-empty
       - `test_compare_same_text()`: POST /api/compare with identical texts, assert unique lists are empty and shared list equals all tokens
       - `test_compare_empty_text()`: POST /api/compare with {"text1": "", "text2": "Hello"}, assert 422 (Pydantic min_length catches it)
